# Reporting Standards

Phase 6. Use this to apply the right reporting checklist and to meet reproducibility norms. Many venues require a checklist at submission. Match the study type to the standard.

## Match the study to the checklist

| Study type | Standard | Scope |
|------------|----------|-------|
| Randomized controlled trial | CONSORT | Trial design, conduct, analysis, reporting |
| Systematic review or meta-analysis | PRISMA | Search, screening, synthesis, bias |
| Observational study | STROBE | Cohort, case control, cross-sectional |
| Diagnostic accuracy study | STARD | Index test against a reference standard |
| Animal research | ARRIVE | Design, animals, procedures, analysis |
| Prediction model | TRIPOD | Development and validation of a model |
| Qualitative study | COREQ, SRQR | Interviews, focus groups, analysis |
| Case report | CARE | Single case description |
| Machine learning in health | TRIPOD-AI, CLAIM | Model development, validation, imaging |
| General data and code | MDAR, FAIR | Materials, design, analysis, reporting |

## CONSORT essentials for trials

- A flow diagram of enrollment, allocation, follow-up, and analysis.
- Trial registration number and the protocol.
- Randomization method, allocation concealment, and blinding.
- Primary and secondary outcomes as pre-specified.
- All randomized participants accounted for.

## PRISMA essentials for systematic reviews

- A flow diagram with records identified, screened, excluded, and included.
- The full search string for at least one database.
- Eligibility criteria and the screening process.
- Risk of bias assessment for included studies.
- Registration on PROSPERO where applicable.

## STROBE essentials for observational studies

- Study design stated in the title or abstract.
- Eligibility, sources, and methods of selection.
- Confounders and how the analysis handled them.
- Participants at each stage with reasons for non-participation.
- Sensitivity analyses.

## Reproducibility and FAIR data

- Findable. A persistent identifier such as a DOI. Rich metadata.
- Accessible. A clear access path and license, even when the data are controlled.
- Interoperable. Standard formats and vocabularies.
- Reusable. Documentation, provenance, and a usage license.

Practical steps:

- Deposit data in a recognized repository with a DOI.
- Release code with a license, a version tag, and a reproduction script.
- Pin software versions and report the environment.
- Provide a data dictionary for every released table.
- State what you cannot share and why.

## Ethics reporting

- Name the approval body and the protocol number.
- State the consent process.
- For human data, state the de-identification standard and the data use agreement.
- Declare funding and competing interests.

## Quality gate for reporting standards

1. The correct checklist is identified for the study type.
2. The required flow diagram is present where the standard demands one.
3. Registration and protocol details are reported where they apply.
4. Data and code meet FAIR practice with identifiers and licenses.
5. Ethics approval, consent, and competing interests are declared.
6. What cannot be shared is stated with the reason.
